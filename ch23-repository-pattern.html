Chapter 23

Repository Pattern

An abstraction layer between business logic and data access that encapsulates queries.

01

What is a Repository?

A repository sits between your business logic and database, handling all data access. Instead of code directly calling SQL or LINQ, it asks the repository for what it needs.

Bad: Direct DbContext Access
// Scattered queries everywhere
var customers = await db.Customers
    .Where(c => c.IsActive && c.Balance > 0)
    .OrderBy(c => c.Name)
    .ToListAsync();

var orders = await db.Orders
    .Where(o => o.CustomerId == customerId)
    .Include(o => o.Items)
    .ToListAsync();
Good: Repository Pattern
// Clear intent, encapsulated logic
var customers = repo.GetActiveCustomers();
var orders = repo.GetOrdersByCustomer(customerId);
Key Idea:

Repository = a collection-like interface to data. Methods return domain objects, not SQL results.

02

Generic Repository (CRUD for Any Entity)

A generic repository provides standard CRUD operations that work with any entity type.

Interface
public interface IRepository<T>
{
    Task<T?> GetByIdAsync(int id);
    Task<IEnumerable<T>> GetAllAsync();
    Task AddAsync(T entity);
    Task UpdateAsync(T entity);
    Task DeleteAsync(int id);
    Task SaveChangesAsync();
}
EF Core Implementation
public class Repository<T> : IRepository<T> where T : class
{
    private readonly DbContext _db;

    public Repository(DbContext db) => _db = db;

    public async Task<T?> GetByIdAsync(int id)
        => await _db.Set<T>().FindAsync(id);

    public async Task<IEnumerable<T>> GetAllAsync()
        => await _db.Set<T>().AsNoTracking().ToListAsync();

    public async Task AddAsync(T entity)
    {
        _db.Set<T>().Add(entity);
        await _db.SaveChangesAsync();
    }

    public async Task UpdateAsync(T entity)
    {
        _db.Set<T>().Update(entity);
        await _db.SaveChangesAsync();
    }

    public async Task DeleteAsync(int id)
    {
        var entity = await GetByIdAsync(id);
        if (entity != null)
        {
            _db.Set<T>().Remove(entity);
            await _db.SaveChangesAsync();
        }
    }

    public async Task SaveChangesAsync()
        => await _db.SaveChangesAsync();
}
Trade-off:

Generic repos are reusable but hide business logic. For specific queries, create domain-specific repositories instead.

03

Specific Repository (Domain-Focused)

A repository for a single entity expresses business intent through method names.

C#
public interface ICustomerRepository
{
    Task<Customer?> GetByIdAsync(int id);
    Task<IEnumerable<Customer>> GetActiveCustomersAsync();
    Task<IEnumerable<Customer>> GetCustomersWithBalanceAboveAsync(decimal amount);
    Task<Customer?> GetByEmailAsync(string email);
    Task AddAsync(Customer customer);
    Task UpdateAsync(Customer customer);
}

public class CustomerRepository : ICustomerRepository
{
    private readonly DbContext _db;

    public CustomerRepository(DbContext db) => _db = db;

    public async Task<IEnumerable<Customer>> GetActiveCustomersAsync()
        => await _db.Customers
            .AsNoTracking()
            .Where(c => c.IsActive)
            .OrderBy(c => c.Name)
            .ToListAsync();

    public async Task<IEnumerable<Customer>> GetCustomersWithBalanceAboveAsync(decimal amount)
        => await _db.Customers
            .AsNoTracking()
            .Where(c => c.Balance >= amount)
            .ToListAsync();

    // ... other methods
}
04

Pagination & Sorting

Real applications need to handle large datasets efficiently.

Repository with Pagination
public class PagedResult<T>
{
    public IEnumerable<T> Items { get; set; }
    public int PageNumber { get; set; }
    public int PageSize { get; set; }
    public int TotalCount { get; set; }
}

public interface ICustomerRepository
{
    Task<PagedResult<Customer>> GetPagedAsync(
        int pageNumber, int pageSize, string orderBy = "Name");
}

public class CustomerRepository : ICustomerRepository
{
    public async Task<PagedResult<Customer>> GetPagedAsync(
        int pageNumber, int pageSize, string orderBy = "Name")
    {
        var query = _db.Customers.AsNoTracking();
        var totalCount = await query.CountAsync();

        var items = await query
            .OrderBy(c => EF.Property<object>(c, orderBy))
            .Skip((pageNumber - 1) * pageSize)
            .Take(pageSize)
            .ToListAsync();

        return new PagedResult<Customer>
        {
            Items = items,
            PageNumber = pageNumber,
            PageSize = pageSize,
            TotalCount = totalCount
        };
    }
}
Database-Level:

Pagination queries also apply in SQL using OFFSET and FETCH to limit large result sets.

05

Specification Pattern (Complex Filters)

For complex filtering, encapsulate query criteria in a reusable specification object.

C#
public abstract class Specification<T>
{
    public Expression<Func<T, bool>> Criteria { get; protected set; }
    public List<Expression<Func<T, object>>> Includes { get; } = new();
    public string OrderBy { get; protected set; }
}

public class ActiveCustomersWithRecentOrdersSpec : Specification<Customer>
{
    public ActiveCustomersWithRecentOrdersSpec(int daysBack = 30)
    {
        var cutoffDate = DateTime.Now.AddDays(-daysBack);

        Criteria = c => c.IsActive
            && c.Orders.Any(o => o.CreatedAt >= cutoffDate);

        Includes.Add(c => c.Orders);
        OrderBy = nameof(Customer.Name);
    }
}

public class CustomerRepository
{
    public async Task<IEnumerable<Customer>> GetBySpecAsync(
        Specification<Customer> spec)
    {
        var query = _db.Customers.AsNoTracking().Where(spec.Criteria);

        foreach (var include in spec.Includes)
            query = query.Include(include);

        return await query.ToListAsync();
    }
}
06

Testing with Mock Repositories

Abstracted repositories make testing easy. Inject a mock repository instead of the real database.

Mock Repository for Testing
public class MockCustomerRepository : ICustomerRepository
{
    private readonly List<Customer> _customers = new();

    public async Task<IEnumerable<Customer>> GetActiveCustomersAsync()
        => await Task.FromResult(
            _customers.Where(c => c.IsActive).OrderBy(c => c.Name)
        );

    public async Task AddAsync(Customer customer)
    {
        _customers.Add(customer);
        await Task.CompletedTask;
    }

    // ... other methods
}

// Usage in test
[Fact]
public async Task GetActiveCustomers_ReturnsOnlyActiveCustomers()
{
    var repo = new MockCustomerRepository();
    await repo.AddAsync(new Customer { Id = 1, Name = "Alice", IsActive = true });
    await repo.AddAsync(new Customer { Id = 2, Name = "Bob", IsActive = false });

    var result = await repo.GetActiveCustomersAsync();

    Assert.Single(result);
    Assert.Equal("Alice", result.First().Name);
}

Key Takeaways

  • Repository = abstraction layer between business logic and data access.
  • Generic repositories provide CRUD for any entity; specific repos express business intent.
  • Method names should reflect domain concepts, not SQL operations.
  • Repositories enable testability through mock implementations.
  • Specification pattern handles complex filtering elegantly.
  • Pagination and sorting should be built into repository queries.
