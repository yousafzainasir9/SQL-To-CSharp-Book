Chapter 05

SELECT & Projections

You get a requirement: "Show me customer names and emails only — not everything." A SQL developer writes SELECT Name, Email. A C# developer writes .Select(c => new { c.Name, c.Email }). Both reduce what's returned — here's how to think about projections in both worlds.

01

Requirement: "Show Only Customer Name and Email"

Business Requirement
API endpoint returns a customer list. Frontend only needs name and email — don't send passwords, addresses, phone numbers, or any other sensitive data.
SQL Developer Thinks Instead of SELECT *, I'll explicitly pick columns: SELECT Name, Email FROM Customers. This reduces network traffic, protects privacy, and makes it obvious what data the API returns.
C# Developer Thinks Use .Select() to project to an anonymous type or DTO: .Select(c => new { c.Name, c.Email }). This reshapes the data before it leaves the database, same benefit as SQL.
SQL — SELECT Columns
SELECT Name, Email
FROM Customers;
LINQ — Method Syntax
var customers = await db.Customers
    .AsNoTracking()
    .Select(c => new { c.Name, c.Email })
    .ToListAsync(ct);
LINQ — Query Syntax
var customers = await (
    from c in db.Customers.AsNoTracking()
    select new { c.Name, c.Email }
).ToListAsync(ct);
Dapper — Direct SQL
await using var conn = new SqlConnection(connectionString);
var customers = await conn.QueryAsync<dynamic>(
    "SELECT Name, Email FROM Customers");
02

Requirement: "Show Full Name and Order Total with Tax"

Business Requirement
Display "FirstName LastName" as one field. Also calculate total with tax (multiply by 1.1). These computed values don't exist in the database — we build them on the fly.
SQL Developer Thinks Use computed columns in the SELECT: FirstName + ' ' + LastName AS FullName, Total * 1.1 AS WithTax. String concatenation and math happen in SQL.
C# Developer Thinks Use string interpolation and math in the .Select(): .Select(c => new { FullName = $"{c.First} {c.Last}", WithTax = c.Total * 1.1m }). Same thing, C# syntax.
SQL — String Concat & Math
SELECT
    FirstName + ' ' + LastName AS FullName,
    Total * 1.1 AS WithTax
FROM Customers;
C# — Interpolation & Math
var result = await db.Customers
    .AsNoTracking()
    .Select(c => new
    {
        FullName = $"{c.First} {c.Last}",
        WithTax = c.Total * 1.1m
    })
    .ToListAsync(ct);
LINQ Query Syntax with let Keyword
var result = await (
    from c in db.Customers.AsNoTracking()
    let fullName = $"{c.First} {c.Last}"
    let withTax = c.Total * 1.1m
    select new { fullName, withTax }
).ToListAsync(ct);
03

Requirement: "List Unique Categories"

Business Requirement
Frontend dropdown filter needs unique category names from products. If we have 1000 products, we shouldn't return 1000 rows — just the unique list.
SQL Developer Thinks SELECT DISTINCT Category FROM Products. The database removes duplicate rows before returning them.
C# Developer Thinks .Select(p => p.Category).Distinct(). LINQ applies deduplication in memory, or better yet, let the database do it with AsNoTracking().
SQL — DISTINCT
SELECT DISTINCT Category
FROM Products
WHERE IsActive = 1
ORDER BY Category;
LINQ — Method Syntax
var categories = await db.Products
    .AsNoTracking()
    .Where(p => p.IsActive)
    .Select(p => p.Category)
    .Distinct()
    .OrderBy(c => c)
    .ToListAsync(ct);
LINQ — Query Syntax
var categories = await (
    from p in db.Products.AsNoTracking()
    where p.IsActive
    select p.Category
).Distinct()
 .OrderBy(c => c)
 .ToListAsync(ct);
Dapper — Direct SQL
await using var conn = new SqlConnection(connectionString);
var categories = (await conn.QueryAsync<string>(
    @"SELECT DISTINCT Category
      FROM Products
      WHERE IsActive = 1
      ORDER BY Category")).ToList();
04

Requirement: "Show Customer Status as Text"

Business Requirement
Instead of showing "1" or "0", show "Active" or "Inactive" in the customer report. Transform boolean data to human-readable text.
SQL Developer Thinks CASE WHEN IsActive = 1 THEN 'Active' ELSE 'Inactive' END converts boolean to text at the database level.
C# Developer Thinks Use a ternary operator in Select: IsActive ? "Active" : "Inactive". Same logic, C# syntax.
SQL — CASE Statement
SELECT
    Id,
    Name,
    CASE WHEN IsActive = 1
         THEN 'Active'
         ELSE 'Inactive'
    END AS Status
FROM Customers;
C# — Ternary in Select
var customers = await db.Customers
    .AsNoTracking()
    .Select(c => new
    {
        c.Id,
        c.Name,
        Status = c.IsActive ? "Active" : "Inactive"
    })
    .ToListAsync(ct);
Dapper — CASE in SQL
await using var conn = new SqlConnection(connectionString);
var customers = await conn.QueryAsync<dynamic>(
    @"SELECT Id, Name,
             CASE WHEN IsActive = 1
                  THEN 'Active'
                  ELSE 'Inactive'
             END AS Status
      FROM Customers");
05

Requirement: "Build an API Response DTO"

Business Requirement
API needs a specific shape: CustomerResponse with Id, FullName, OrderCount, and LastOrderDate. Project raw database tables into this structured response.
SQL Developer Thinks SELECT c.Id, c.FirstName + ' ' + c.LastName AS FullName, COUNT(o.Id) AS OrderCount, MAX(o.CreatedAt) AS LastOrderDate FROM Customers c LEFT JOIN Orders o ON c.Id = o.CustomerId GROUP BY c.Id, c.FirstName, c.LastName.
C# Developer Thinks Project to a named DTO or record type in the .Select(): .Select(c => new CustomerResponse { Id = c.Id, FullName = $"{c.First} {c.Last}", ... }). Strong typing, reusable across endpoints.
C# — Record Definition (Immutable DTO)
public record CustomerResponse(
    int Id,
    string FullName,
    int OrderCount,
    DateTime? LastOrderDate
);
C# — EF Projection to Record
var customers = await db.Customers
    .AsNoTracking()
    .Select(c => new CustomerResponse(
        Id: c.Id,
        FullName: $"{c.First} {c.Last}",
        OrderCount: c.Orders.Count(),
        LastOrderDate: c.Orders
            .Max(o => (DateTime?)o.CreatedAt)
    ))
    .ToListAsync(ct);
SQL — Join with Aggregation
SELECT
    c.Id,
    c.FirstName + ' ' + c.LastName AS FullName,
    COUNT(o.Id) AS OrderCount,
    MAX(o.CreatedAt) AS LastOrderDate
FROM Customers c
LEFT JOIN Orders o ON c.Id = o.CustomerId
GROUP BY c.Id, c.FirstName, c.LastName
ORDER BY c.Id;
Dapper — Map to DTO
await using var conn = new SqlConnection(connectionString);
var customers = await conn.QueryAsync<CustomerResponse>(
    @"SELECT c.Id,
             c.FirstName + ' ' + c.LastName AS FullName,
             COUNT(o.Id) AS OrderCount,
             MAX(o.CreatedAt) AS LastOrderDate
      FROM Customers c
      LEFT JOIN Orders o ON c.Id = o.CustomerId
      GROUP BY c.Id, c.FirstName, c.LastName
      ORDER BY c.Id");
06

Requirement: "Show Each Customer with Their Latest Order"

Business Requirement
Customer list shows name + most recent order date and amount. We need nested data: customer info with embedded latest order details.
SQL Developer Thinks Use a subquery or CROSS APPLY to grab the latest order for each customer. Most recent order — sorted DESC, take the first one.
C# Developer Thinks Nested .Select() inside the projection. Inside the outer .Select(), use .OrderByDescending().FirstOrDefault() to get the latest order for each customer.
SQL — Subquery for Latest Order
SELECT
    c.Id,
    c.Name,
    (SELECT TOP 1 o.OrderDate
     FROM Orders o
     WHERE o.CustomerId = c.Id
     ORDER BY o.OrderDate DESC) AS LatestOrderDate,
    (SELECT TOP 1 o.Total
     FROM Orders o
     WHERE o.CustomerId = c.Id
     ORDER BY o.OrderDate DESC) AS LatestOrderTotal
FROM Customers c
ORDER BY c.Id;
C# — Nested LINQ Projection
var customers = await db.Customers
    .AsNoTracking()
    .Select(c => new
    {
        c.Id,
        c.Name,
        LatestOrder = c.Orders
            .OrderByDescending(o => o.OrderDate)
            .Select(o => new { o.OrderDate, o.Total })
            .FirstOrDefault()
    })
    .ToListAsync(ct);
Dapper — Multi-Map with Related Data
await using var conn = new SqlConnection(connectionString);
var query = @"
SELECT c.Id, c.Name, o.OrderDate, o.Total
FROM Customers c
LEFT JOIN (
    SELECT CustomerId, OrderDate, Total,
           ROW_NUMBER() OVER (PARTITION BY CustomerId
                             ORDER BY OrderDate DESC) AS rn
    FROM Orders
) o ON c.Id = o.CustomerId AND o.rn = 1
ORDER BY c.Id";

var customers = (await conn.QueryAsync<dynamic>(query))
    .GroupBy(x => x.Id)
    .Select(g => new
    {
        Id = g.Key,
        Name = g.First().Name,
        LatestOrderDate = g.First().OrderDate,
        LatestOrderTotal = g.First().Total
    })
    .ToList();
