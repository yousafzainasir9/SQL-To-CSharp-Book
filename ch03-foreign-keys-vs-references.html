Chapter 03

Foreign Keys vs References

You get a requirement: "Orders must belong to a customer." A SQL developer adds a FOREIGN KEY constraint. A C# developer adds a navigation property. Both enforce the same relationship — here's how they connect.

01

Requirement: "Every Order Must Belong to a Customer"

Business Requirement
When creating an order, it must be linked to an existing customer. No orphan orders allowed. If a user tries to create an order with an invalid customer ID, the system should reject it.
SQL Developer Thinks I'll add a CustomerId column with FOREIGN KEY REFERENCES Customers(Id). The database rejects inserts and updates with invalid customer IDs automatically. The constraint is enforced at the database level, no matter what.
C# Developer Thinks I'll add a CustomerId property (the foreign key value) and a Customer navigation property (the actual object). EF links them together, validates before saving, and lets me access customer.Name directly without manual JOINs.
SQL — Create Tables with FK
CREATE TABLE Customers (
    Id    INT IDENTITY(1,1) PRIMARY KEY,
    Name  NVARCHAR(100) NOT NULL
);

CREATE TABLE Orders (
    Id         INT IDENTITY(1,1) PRIMARY KEY,
    OrderDate  DATETIME2 NOT NULL DEFAULT GETDATE(),
    Total      DECIMAL(10,2) NOT NULL,
    CustomerId INT NOT NULL
               FOREIGN KEY REFERENCES Customers(Id)
);
C# — Entity Classes
public class Customer
{
    public int Id { get; set; }
    public string Name { get; set; } = "";

    // Navigation: collection of this customer's orders
    public ICollection<Order> Orders { get; set; } = [];
}

public class Order
{
    public int Id { get; set; }
    public DateTime OrderDate { get; set; }
    public decimal Total { get; set; }

    // FK value
    public int CustomerId { get; set; }

    // Navigation property
    public Customer Customer { get; set; } = null!;
}
SQL — Get Customer Name with JOIN
SELECT o.Id, o.OrderDate, o.Total, c.Name
FROM Orders o
JOIN Customers c ON o.CustomerId = c.Id
WHERE o.Id = 42;
C# — Get Customer Name with Include
var order = await db.Orders
    .AsNoTracking()
    .Include(o => o.Customer)
    .Where(o => o.Id == 42)
    .FirstOrDefaultAsync(ct);

// Now we can access:
var customerName = order?.Customer.Name;
Dapper — Manual Multi-Mapping with splitOn
await using var conn = new SqlConnection(connectionString);
var orders = await conn.QueryAsync<Order, Customer, Order>(
    @"SELECT o.Id, o.OrderDate, o.Total, o.CustomerId,
             c.Id, c.Name
      FROM Orders o
      JOIN Customers c ON o.CustomerId = c.Id
      WHERE o.Id = @OrderId",
    (o, c) => { o.Customer = c; return o; },
    new { OrderId = 42 },
    splitOn: "Id");

var result = orders.FirstOrDefault();
02

Requirement: "What Happens When We Delete a Customer?"

Business Requirement
If a customer account is deleted, what happens to their orders? The business requirement: keep the orders but clear the customer link (they become orphaned, but not deleted). Different scenarios need different cascade rules.
SQL Developer Thinks I use ON DELETE SET NULL — when the parent is deleted, the FK becomes NULL instead of deleting the child rows. Alternatively: CASCADE (delete children too), RESTRICT (prevent deletion), SET DEFAULT (set to a default value).
C# Developer Thinks In Fluent API, I call .OnDelete(DeleteBehavior.SetNull). EF enforces this logic when SaveChanges() is called. I need to match the database constraint behavior to avoid conflicts.
ScenarioSQL SyntaxEF DeleteBehaviorEffect
Keep children, clear FKON DELETE SET NULLSetNullCustomerId becomes NULL, order survives
Delete children tooON DELETE CASCADECascadeDeleting customer deletes all their orders
Prevent deletionON DELETE RESTRICT or NO ACTIONRestrictCannot delete customer if orders exist
Set to defaultON DELETE SET DEFAULTSetDefaultFK set to a default customer ID
No enforcement—NoActionEF doesn't enforce, database decides
SQL — ALTER TABLE
-- Option 1: Keep orders, clear FK
ALTER TABLE Orders
DROP CONSTRAINT FK_Orders_Customers;

ALTER TABLE Orders ADD CONSTRAINT FK_Orders_Customers
    FOREIGN KEY (CustomerId)
    REFERENCES Customers(Id)
    ON DELETE SET NULL;

-- Option 2: Delete orders too
ALTER TABLE Orders ADD CONSTRAINT FK_Orders_Customers
    FOREIGN KEY (CustomerId)
    REFERENCES Customers(Id)
    ON DELETE CASCADE;
C# — Fluent API Configuration
// Option 1: Keep orders, clear FK
modelBuilder.Entity<Order>(e =>
{
    e.HasOne(o => o.Customer)
     .WithMany(c => c.Orders)
     .HasForeignKey(o => o.CustomerId)
     .OnDelete(DeleteBehavior.SetNull);
});

// Option 2: Delete orders too
modelBuilder.Entity<Order>(e =>
{
    e.HasOne(o => o.Customer)
     .WithMany(c => c.Orders)
     .HasForeignKey(o => o.CustomerId)
     .OnDelete(DeleteBehavior.Cascade);
});
03

Requirement: "Some Orders Can Be Anonymous"

Business Requirement
Guest checkout allows orders without a customer account. Regular checkout requires one. So CustomerId can be NULL for guest orders, but NOT NULL for registered customers.
SQL Developer Thinks Simple: Required = NOT NULL, Optional = NULL. The database enforces it. If a column is NULL, guests can check out. If NOT NULL, registration is required.
C# Developer Thinks Required = int CustomerId (non-nullable). Optional = int? CustomerId (nullable). If it's optional, the navigation property must also be nullable: Customer? instead of Customer.
SQL — Required FK (NOT NULL)
-- Registered customer (FK required)
CREATE TABLE Orders (
    Id         INT IDENTITY(1,1) PRIMARY KEY,
    CustomerId INT NOT NULL
               FOREIGN KEY REFERENCES Customers(Id),
    Total      DECIMAL(10,2) NOT NULL
);
C# — Required FK (Non-Nullable)
public class Order
{
    public int Id { get; set; }

    // Required: must have a customer
    public int CustomerId { get; set; }
    public Customer Customer { get; set; } = null!;

    public decimal Total { get; set; }
}
SQL — Optional FK (NULL)
-- Guest order (FK optional)
CREATE TABLE Orders (
    Id         INT IDENTITY(1,1) PRIMARY KEY,
    CustomerId INT NULL
               FOREIGN KEY REFERENCES Customers(Id),
    Total      DECIMAL(10,2) NOT NULL
);
C# — Optional FK (Nullable)
public class Order
{
    public int Id { get; set; }

    // Optional: guest can check out
    public int? CustomerId { get; set; }
    public Customer? Customer { get; set; }

    public decimal Total { get; set; }
}
04

Requirement: "Employees Report to a Manager"

Business Requirement
Every employee has a manager (who is also an employee). The CEO has no manager. Build a hierarchy where employees can reference other employees.
SQL Developer Thinks ManagerId INT NULL FOREIGN KEY REFERENCES Employees(Id) — same table, self-referencing. Perfect for hierarchies.
C# Developer Thinks Manager navigation property + Subordinates collection on the same Employee class. Query with .Include(e => e.Manager).Include(e => e.Subordinates) to load the tree structure.
SQL — Self-Referencing FK
CREATE TABLE Employees (
    Id        INT IDENTITY(1,1) PRIMARY KEY,
    Name      NVARCHAR(100) NOT NULL,
    Title     NVARCHAR(100) NOT NULL,
    ManagerId INT NULL
              FOREIGN KEY REFERENCES Employees(Id),
    HireDate  DATETIME2 NOT NULL DEFAULT GETDATE()
);
C# — Self-Referencing Navigation
public class Employee
{
    public int Id { get; set; }
    public string Name { get; set; } = "";
    public string Title { get; set; } = "";
    public DateTime HireDate { get; set; }

    // FK: the manager's ID (NULL for CEO)
    public int? ManagerId { get; set; }

    // Navigation: the manager object
    public Employee? Manager { get; set; }

    // Navigation: all direct reports
    public ICollection<Employee> Subordinates { get; set; } = [];
}
C# — Query Employee Hierarchy with Includes
// Load an employee with their manager and subordinates
var employee = await db.Employees
    .AsNoTracking()
    .Include(e => e.Manager)
    .Include(e => e.Subordinates)
    .FirstOrDefaultAsync(e => e.Id == 5, ct);

// Now we can access:
var managerName = employee?.Manager?.Name;  // Their boss
var reportCount = employee?.Subordinates.Count;  // How many report to them
SQL — Find All Reports (Recursive CTE)
-- Find all employees under manager ID 1 (recursively)
WITH EmployeeHierarchy AS (
    SELECT Id, Name, ManagerId, 0 AS Level
    FROM Employees
    WHERE Id = 1

    UNION ALL

    SELECT e.Id, e.Name, e.ManagerId, eh.Level + 1
    FROM Employees e
    INNER JOIN EmployeeHierarchy eh ON e.ManagerId = eh.Id
)
SELECT * FROM EmployeeHierarchy;
05

Requirement: "Configure All Relationships in One Place"

Business Requirement
Dev team wants all FK configuration centralized in OnModelCreating(), not scattered across entities as attributes. Single source of truth for all relationships.
SQL Developer Thinks All constraints defined in CREATE TABLE or ALTER TABLE scripts. One place, all relationships documented.
C# Developer Thinks All configured in OnModelCreating() with Fluent API. HasOne/WithMany/HasForeignKey/OnDelete — everything in one DbContext method. Cleaner entities, all config centralized.
C# — Complete Fluent API Configuration for Orders & Employees
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    // Order -> Customer (many-to-one, optional)
    modelBuilder.Entity<Order>(e =>
    {
        e.HasKey(o => o.Id);

        e.Property(o => o.OrderDate)
         .HasDefaultValueSql("GETDATE()");

        e.Property(o => o.Total)
         .HasPrecision(10, 2);

        e.HasOne(o => o.Customer)
         .WithMany(c => c.Orders)
         .HasForeignKey(o => o.CustomerId)
         .OnDelete(DeleteBehavior.SetNull);
    });

    // Customer
    modelBuilder.Entity<Customer>(e =>
    {
        e.HasKey(c => c.Id);
        e.Property(c => c.Name).IsRequired().HasMaxLength(100);
    });

    // Employee -> Employee (self-ref, optional)
    modelBuilder.Entity<Employee>(e =>
    {
        e.HasKey(e => e.Id);

        e.Property(e => e.Name).IsRequired().HasMaxLength(100);
        e.Property(e => e.Title).IsRequired().HasMaxLength(100);
        e.Property(e => e.HireDate)
         .HasDefaultValueSql("GETDATE()");

        // Self-reference: manager
        e.HasOne(e => e.Manager)
         .WithMany(e => e.Subordinates)
         .HasForeignKey(e => e.ManagerId)
         .OnDelete(DeleteBehavior.Restrict);
    });
}
06

Requirement: "Insert and Query with Foreign Keys"

Business Requirement
Create a new order for an existing customer. Then query all orders with customer names in one efficient operation.

Creating an Order for an Existing Customer

SQL — INSERT
INSERT INTO Orders (
    OrderDate, CustomerId, Total
)
VALUES (
    GETDATE(), 42, 199.99
);

SELECT SCOPE_IDENTITY() AS Id;
EF Core — Add & SaveChanges
var order = new Order
{
    OrderDate = DateTime.Now,
    CustomerId = 42,
    Total = 199.99m
};

db.Orders.Add(order);
await db.SaveChangesAsync(ct);

// order.Id is now set
Dapper — ExecuteScalarAsync
await using var conn = new SqlConnection(
    connectionString);
var id = await conn.ExecuteScalarAsync<int>(
    @"INSERT INTO Orders (
        OrderDate, CustomerId, Total
      )
      VALUES (GETDATE(), @CustId, @Total);
      SELECT SCOPE_IDENTITY()",
    new { CustId = 42, Total = 199.99m });

Query Orders with Customer Names

SQL — JOIN
SELECT o.Id, o.OrderDate, o.Total,
       c.Name AS CustomerName
FROM Orders o
LEFT JOIN Customers c ON o.CustomerId = c.Id
ORDER BY o.OrderDate DESC;
LINQ — Include & Select
var orders = await db.Orders
    .AsNoTracking()
    .Include(o => o.Customer)
    .OrderByDescending(o => o.OrderDate)
    .Select(o => new
    {
        o.Id,
        o.OrderDate,
        o.Total,
        CustomerName = o.Customer!.Name
    })
    .ToListAsync(ct);
Dapper — Multi-Mapping
await using var conn = new SqlConnection(
    connectionString);
var results = await conn.QueryAsync<
    OrderDto, CustomerDto, OrderDto>(
    @"SELECT o.Id, o.OrderDate, o.Total,
             c.Id, c.Name
      FROM Orders o
      LEFT JOIN Customers c ON o.CustomerId = c.Id
      ORDER BY o.OrderDate DESC",
    (o, c) => { o.CustomerName = c?.Name;
               return o; },
    splitOn: "Id");

Query with Filtering

SQL — Find Orders by Customer
SELECT o.Id, o.OrderDate, o.Total
FROM Orders o
WHERE o.CustomerId = 42
  AND o.OrderDate >= DATEADD(MONTH, -1, GETDATE())
ORDER BY o.OrderDate DESC;
LINQ — Where & OrderByDescending
var cutoff = DateTime.Now.AddMonths(-1);
var orders = await db.Orders
    .AsNoTracking()
    .Where(o => o.CustomerId == 42
             && o.OrderDate >= cutoff)
    .OrderByDescending(o => o.OrderDate)
    .ToListAsync(ct);
SQL — Count Orders by Customer
SELECT c.Id, c.Name, COUNT(o.Id) AS OrderCount
FROM Customers c
LEFT JOIN Orders o ON c.Id = o.CustomerId
GROUP BY c.Id, c.Name
HAVING COUNT(o.Id) > 0
ORDER BY COUNT(o.Id) DESC;
LINQ — GroupBy & Select
var stats = await db.Customers
    .AsNoTracking()
    .GroupBy(c => c.Id)
    .Where(g => g.SelectMany(c => c.Orders)
                  .Any())
    .Select(g => new
    {
        CustomerId = g.Key,
        OrderCount = g.SelectMany(c => c.Orders)
                      .Count()
    })
    .OrderByDescending(x => x.OrderCount)
    .ToListAsync(ct);
