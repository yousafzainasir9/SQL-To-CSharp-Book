Connection & Queries with Dapper

Dapper is a lightweight micro-ORM that maps SQL results to C# objects with minimal overhead.

01

What is Dapper?

Dapper is a micro-ORM — a thin wrapper around IDbConnection that maps SQL query results to C# objects. You write SQL; Dapper handles object mapping.

SQL (You Write This) 
SELECT Id, Name, Email
FROM Customers
WHERE Department = @Department
ORDER BY CreatedAt DESC
C# (Dapper Maps It) 
var customers = connection
    .Query<Customer>(sql,
        new { Department = "Sales" })
    .ToList();
Why Dapper? Speed and control. No change tracking overhead. No N+1 queries unless you write them. Minimal configuration.
02

Installation & Setup

Add Dapper via NuGet. Zero dependencies (except your database driver).

Package Manager Console 
Install-Package Dapper

Create a connection and start querying:

C# 
public async Task<Customer> GetCustomerAsync(int id)
{
    await using var connection = new SqlConnection(connectionString);
    var sql = "SELECT * FROM Customers WHERE Id = @Id";
    return await connection
        .QueryFirstOrDefaultAsync<Customer>(sql, new { Id = id });
}
03

Parameterized Queries & SQL Injection

Always use parameters (e.g., @Name) to prevent SQL injection. Dapper automatically handles parameter binding from anonymous objects.

Bad: String Concatenation 
// DANGEROUS - SQL Injection risk!
var sql = $"SELECT * FROM Customers WHERE Name = '{name}'";
var result = connection.Query<Customer>(sql);
Good: Parameterized 
var sql = "SELECT * FROM Customers WHERE Name = @Name";
var result = connection.Query<Customer>(sql,
    new { Name = name });
Parameter Rule: Always use @ParameterName in SQL and pass an object with matching properties to Dapper.
04

Query Methods: Query, First, Single, FirstOrDefault

Dapper provides multiple methods depending on how many rows you expect:

Method Returns If No Match
Query<T> IEnumerable<T> Empty collection
QueryFirst<T> T (first row) Exception
QueryFirstOrDefault<T> T or null null
QuerySingle<T> T (exactly one row) Exception
QuerySingleOrDefault<T> T or null null
QueryScalar<T> Single value null or default
C# Examples 
// Get all customers
var all = await connection.QueryAsync<Customer>(
    "SELECT * FROM Customers");

// Get first customer or null
var one = await connection.QueryFirstOrDefaultAsync<Customer>(
    "SELECT * FROM Customers WHERE Id = @Id",
    new { Id = 1 });

// Get a single count value
var count = await connection.QueryScalarAsync<int>(
    "SELECT COUNT(*) FROM Customers");
05

INSERT, UPDATE, DELETE with Execute

Use Execute for INSERT, UPDATE, DELETE. It returns the number of rows affected.

C# 
// INSERT
var insertSql = @"INSERT INTO Customers (Name, Email, CreatedAt)
                  VALUES (@Name, @Email, GETDATE())";
int rowsInserted = await connection.ExecuteAsync(insertSql,
    new { Name = "Alice", Email = "alice@example.com" });

// UPDATE
var updateSql = "UPDATE Customers SET Email = @Email WHERE Id = @Id";
int rowsUpdated = await connection.ExecuteAsync(updateSql,
    new { Email = "newemail@example.com", Id = 1 });

// DELETE
var deleteSql = "DELETE FROM Customers WHERE Id = @Id";
int rowsDeleted = await connection.ExecuteAsync(deleteSql, new { Id = 1 });
Transactions: Wrap multiple operations in a transaction for safety.
06

Async Queries & Transactions

Dapper supports async/await. Use QueryAsync, ExecuteAsync, etc. for non-blocking operations.

Async Query 
public async Task<Customer> GetCustomerAsync(int id)
{
    await using var connection = new SqlConnection(connectionString);
    var sql = "SELECT * FROM Customers WHERE Id = @Id";
    return await connection
        .QueryFirstOrDefaultAsync<Customer>(sql, new { Id = id });
}
Transaction Example 
await using var connection = new SqlConnection(connectionString);
connection.Open();
using var transaction = connection.BeginTransaction();
try
{
    await connection.ExecuteAsync("INSERT INTO Customers (Name) VALUES (@Name)",
        new { Name = "Bob" }, transaction);
    await connection.ExecuteAsync("INSERT INTO Orders (CustomerId) VALUES (@Id)",
        new { Id = 1 }, transaction);
    transaction.Commit();
}
catch
{
    transaction.Rollback();
    throw;
}

Key Takeaways

  • Dapper is a lightweight micro-ORM for SQL-first developers.
  • Always use parameterized queries to prevent SQL injection.
  • Choose the right query method: Query, QueryFirst, QuerySingle, etc.
  • Use Execute for INSERT, UPDATE, DELETE operations.
  • Async variants (QueryAsync, ExecuteAsync) support modern async patterns.
  • Wrap multi-step operations in transactions for data consistency.
