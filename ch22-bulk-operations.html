Chapter 22

Bulk Operations with Dapper

Insert, update, and delete thousands of records efficiently using batch operations and SqlBulkCopy. Choose the right tool for your scale.

01

Batch Inserts with Execute

Insert multiple rows by passing a collection to Execute. Dapper loops internally and batches efficiently.

C#
var customers = new List<Customer>
{
    new { FirstName = "Alice", LastName = "Smith" },
    new { FirstName = "Bob", LastName = "Jones" },
    new { FirstName = "Charlie", LastName = "Brown" }
};

var sql = @"INSERT INTO Customers (FirstName, LastName, CreatedAt)
            VALUES (@FirstName, @LastName, GETDATE())";

int rowsInserted = await connection.ExecuteAsync(sql, customers);
Console.WriteLine($"Inserted {rowsInserted} rows");
Performance: Much faster than looping and executing one-by-one. Dapper batches execution internally.
02

Batch Updates

Update multiple rows the same way: pass a collection to Execute.

C#
var updates = new List<object>
{
    new { FirstName = "Alice Updated", Id = 1 },
    new { FirstName = "Bob Updated", Id = 2 },
    new { FirstName = "Charlie Updated", Id = 3 }
};

var sql = "UPDATE Customers SET FirstName = @FirstName WHERE Id = @Id";
int rowsUpdated = await connection.ExecuteAsync(sql, updates);
Console.WriteLine($"Updated {rowsUpdated} rows");
Batch syntax: Same as batch insert—pass a collection and Dapper executes each item.
03

SqlBulkCopy for Large Datasets

For 10,000+ rows, use SqlBulkCopy (SQL Server only). It bypasses normal INSERT logic for speed.

C# — Convert List to DataTable
public static DataTable ToDataTable<T>(List<T> items)
{
    var dt = new DataTable(typeof(T).Name);
    var props = typeof(T).GetProperties();

    foreach (var prop in props)
        dt.Columns.Add(prop.Name, Nullable.GetUnderlyingType(prop.PropertyType) ?? prop.PropertyType);

    foreach (var item in items)
    {
        var row = dt.NewRow();
        foreach (var prop in props)
            row[prop.Name] = prop.GetValue(item) ?? DBNull.Value;
        dt.Rows.Add(row);
    }
    return dt;
}
C# — Bulk Insert
var customers = GenerateLargeList(); // 100,000+ rows
var dt = ToDataTable(customers);

await using var connection = new SqlConnection(connectionString);
connection.Open();
using var bulkCopy = new SqlBulkCopy(connection)
{
    DestinationTableName = "Customers"
};
await bulkCopy.WriteToServerAsync(dt);

Console.WriteLine($"Bulk inserted {dt.Rows.Count} rows");
When to use: 10,000+ rows: use SqlBulkCopy. Hundreds: use Execute batches. Tens: loop normally.
04

Batch Deletes

Delete multiple rows by passing a collection of IDs.

C#
var idsToDelete = new List<int> { 1, 2, 3, 4, 5 };

var sql = "DELETE FROM Customers WHERE Id = @Id";
int rowsDeleted = await connection.ExecuteAsync(sql,
    idsToDelete.Select(id => new { Id = id }));

Console.WriteLine($"Deleted {rowsDeleted} rows");
Alternative: For many IDs, use a WHERE...IN clause: DELETE FROM Customers WHERE Id IN (@Ids).
05

Transactions & Bulk Operations

Wrap bulk operations in a transaction for safety. If any batch fails, rollback all changes.

C#
await using var connection = new SqlConnection(connectionString);
connection.Open();
using var transaction = connection.BeginTransaction();
try
{
    // Insert customers
    var insertSql = @"INSERT INTO Customers (FirstName, LastName)
                      VALUES (@FirstName, @LastName)";
    await connection.ExecuteAsync(insertSql, customers, transaction);

    // Insert related orders
    var orderSql = @"INSERT INTO Orders (CustomerId, Total)
                     VALUES (@CustomerId, @Total)";
    await connection.ExecuteAsync(orderSql, orders, transaction);

    transaction.Commit();
}
catch
{
    transaction.Rollback();
    throw;
}
Safety: Pass transaction as the 3rd parameter to Execute. If any step fails, rollback everything.
06

Performance Comparison

Choose the right tool based on data volume. Loop for small batches, Execute for medium, SqlBulkCopy for large.

Approximate Times (1 million rows)
// Loop with individual INSERT: ~80 seconds
foreach (var customer in customers)
    await connection.ExecuteAsync(insertSql, customer);

// Batch Execute (1000 rows at a time): ~3 seconds
for (int i = 0; i < customers.Count; i += 1000)
    await connection.ExecuteAsync(insertSql, customers.Skip(i).Take(1000));

// SqlBulkCopy: ~0.5 seconds
await bulkCopy.WriteToServerAsync(dt);
Scale matters: Use SqlBulkCopy only when performance is critical. For most apps, Execute batches are sufficient.

Key Takeaways

  • Use Execute(sql, collection) for batch INSERT/UPDATE/DELETE.
  • Pass transactions to Execute: Execute(sql, data, transaction).
  • SqlBulkCopy is fastest for 10,000+ rows but requires DataTable conversion.
  • Batch executions loop internally; loop manually only for small batches.
  • Wrap bulk ops in transactions to ensure all-or-nothing consistency.
  • Choose based on data volume: loop for tens, Execute for hundreds, SqlBulkCopy for thousands+.
