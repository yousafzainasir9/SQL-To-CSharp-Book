Chapter 24

Unit of Work

Manages transaction boundaries and coordinates multiple repositories.

01

What is Unit of Work?

Unit of Work coordinates multiple repositories and ensures all changes are saved in a single transaction. If any fails, everything rolls back.

Without UoW: Fragmented Saves
var customer = new Customer { Name = "Alice" };
await customerRepo.AddAsync(customer);

var order = new Order { CustomerId = customer.Id };
await orderRepo.AddAsync(order);

var item = new OrderItem { OrderId = order.Id };
await itemRepo.AddAsync(item);

// Multiple SaveChangesAsync calls = multiple transactions
// If item fails, customer and order already saved
With UoW: Atomic Transaction
var customer = new Customer { Name = "Alice" };
uow.Customers.Add(customer);

var order = new Order { CustomerId = customer.Id };
uow.Orders.Add(order);

var item = new OrderItem { OrderId = order.Id };
uow.OrderItems.Add(item);

// Single SaveAsync() = single transaction
await uow.SaveAsync();
// All or nothing
Key Idea:

Unit of Work = single transaction containing changes from multiple repositories. All succeed or all fail.

02

Implementation Pattern

Create an interface that exposes repositories and manages a single transaction.

Interface
public interface IUnitOfWork : IDisposable
{
    ICustomerRepository Customers { get; }
    IOrderRepository Orders { get; }
    IProductRepository Products { get; }

    Task<int> SaveAsync();
    Task BeginTransactionAsync();
    Task CommitAsync();
    Task RollbackAsync();
}

public class UnitOfWork : IUnitOfWork
{
    private readonly DbContext _db;
    private IDbContextTransaction _transaction;

    public ICustomerRepository Customers { get; private set; }
    public IOrderRepository Orders { get; private set; }
    public IProductRepository Products { get; private set; }

    public UnitOfWork(DbContext db)
    {
        _db = db;
        Customers = new CustomerRepository(db);
        Orders = new OrderRepository(db);
        Products = new ProductRepository(db);
    }

    public async Task<int> SaveAsync(CancellationToken cancellationToken = default)
        => await _db.SaveChangesAsync(cancellationToken);

    public async Task BeginTransactionAsync(CancellationToken cancellationToken = default)
        => _transaction = await _db.Database.BeginTransactionAsync(cancellationToken);

    public async Task CommitAsync(CancellationToken cancellationToken = default)
    {
        await _db.SaveChangesAsync(cancellationToken);
        await _transaction.CommitAsync(cancellationToken);
    }

    public async Task RollbackAsync(CancellationToken cancellationToken = default)
        => await _transaction.RollbackAsync(cancellationToken);

    public void Dispose()
    {
        _transaction?.Dispose();
        _db.Dispose();
    }
}
03

Using Unit of Work

Inject UoW into services to coordinate complex operations across multiple repositories.

Service Using Unit of Work
public class OrderService
{
    private readonly IUnitOfWork _uow;

    public OrderService(IUnitOfWork uow) => _uow = uow;

    public async Task CreateOrderAsync(
        int customerId, List<OrderItemDto> items, CancellationToken cancellationToken = default)
    {
        await _uow.BeginTransactionAsync(cancellationToken);

        try
        {
            // Verify customer exists
            var customer = await _uow.Customers.GetByIdAsync(customerId, cancellationToken);
            if (customer is null)
                throw new InvalidOperationException("Customer not found");

            // Create order
            var order = new Order
            {
                CustomerId = customerId,
                CreatedAt = DateTime.Now,
                Total = 0
            };
            await _uow.Orders.AddAsync(order, cancellationToken);
            await _uow.SaveAsync(cancellationToken);

            // Add items
            decimal total = 0;
            foreach (var itemDto in items)
            {
                var product = await _uow.Products.GetByIdAsync(itemDto.ProductId, cancellationToken);
                if (product is null)
                    throw new InvalidOperationException($"Product {itemDto.ProductId} not found");

                var item = new OrderItem
                {
                    OrderId = order.Id,
                    ProductId = product.Id,
                    Quantity = itemDto.Quantity,
                    Price = product.Price
                };
                await _uow.OrderItems.AddAsync(item, cancellationToken);
                total += product.Price * itemDto.Quantity;
            }

            // Update order total
            order.Total = total;
            await _uow.Orders.UpdateAsync(order, cancellationToken);

            await _uow.CommitAsync(cancellationToken);
        }
        catch
        {
            await _uow.RollbackAsync();
            throw;
        }
    }
}
Benefit:

If any step fails, the entire operation rolls back. No partial data corruption.

04

DbContext Is Already a Unit of Work

EF Core's DbContext implements Unit of Work natively. SaveChangesAsync() commits all pending changes in a transaction.

DbContext Acts as UoW
public class SimpleOrderService
{
    private readonly DbContext _db;

    public async Task CreateOrderAsync(
        int customerId, List<OrderItemDto> items, CancellationToken cancellationToken = default)
    {
        var customer = await _db.Customers.FindAsync(new object[] { customerId }, cancellationToken);
        if (customer is null)
            throw new InvalidOperationException("Customer not found");

        var order = new Order { CustomerId = customerId, CreatedAt = DateTime.Now };
        _db.Orders.Add(order);

        foreach (var itemDto in items)
        {
            var product = await _db.Products.FindAsync(new object[] { itemDto.ProductId }, cancellationToken);
            _db.OrderItems.Add(new OrderItem
            {
                OrderId = order.Id,
                ProductId = product.Id,
                Quantity = itemDto.Quantity
            });
        }

        // Single transaction
        await _db.SaveChangesAsync(cancellationToken);
    }
}
When to use explicit UoW:

For complex multi-repository operations or when you need finer control over transaction lifecycle.

05

Registering in DI Container

Configure Unit of Work and repositories in the service collection.

Startup/Program.cs
public void ConfigureServices(IServiceCollection services)
{
    services.AddDbContext<AppDbContext>(options =>
        options.UseSqlServer(Configuration.GetConnectionString("DefaultConnection")));

    // Register repositories
    services.AddScoped<ICustomerRepository, CustomerRepository>();
    services.AddScoped<IOrderRepository, OrderRepository>();
    services.AddScoped<IProductRepository, ProductRepository>();

    // Register Unit of Work
    services.AddScoped<IUnitOfWork, UnitOfWork>();

    // Services that use UoW
    services.AddScoped<IOrderService, OrderService>();
}

// Usage in controller
[ApiController]
[Route("api/orders")]
public class OrderController
{
    private readonly IOrderService _orderService;

    public OrderController(IOrderService orderService)
        => _orderService = orderService;

    [HttpPost]
    public async Task<IActionResult> CreateOrder(CreateOrderRequest request, CancellationToken cancellationToken)
    {
        await _orderService.CreateOrderAsync(request.CustomerId, request.Items, cancellationToken);
        return Ok();
    }
}

Key Takeaways

  • Unit of Work coordinates multiple repositories in a single transaction.
  • All changes commit together or all rollback on error.
  • DbContext is already a Unit of Work pattern implementation.
  • Explicit UoW useful for complex multi-repository operations.
  • Transaction control prevents partial data corruption.
  • Pair UoW with dependency injection for clean architecture.
