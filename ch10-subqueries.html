Chapter 10

Subqueries

SQL uses subqueries — nested SELECT statements — to solve complex problems in one query. C# uses nested LINQ expressions and query composition to achieve the same result. Both embed one query inside another to filter, count, or aggregate data.

01

Requirement: "Show Each Customer with Their Order Count"

Business Requirement
List all customers with a column showing how many orders each has placed. The order count must come from a subquery that counts matching orders.
SQL Developer Thinks I'll use a scalar subquery in the SELECT clause. For each customer, I run a COUNT subquery that filters to that customer's orders. It's called "scalar" because it returns a single value per row.
C# Developer Thinks I'll use .Select() to project a new object with Name and a calculated OrderCount. The Count() method on the Orders collection gives me the same result as the SQL subquery.
SQL — Scalar Subquery
SELECT
    c.Name,
    (SELECT COUNT(*)
     FROM Orders o
     WHERE o.CustomerId = c.Id) AS OrderCount
FROM Customers c;
C# — LINQ Projection
var result = await db.Customers
    .AsNoTracking()
    .Select(c => new
    {
        c.Name,
        OrderCount = c.Orders.Count()
    })
    .ToListAsync(ct);
02

Requirement: "Find Customers Who Have Placed Orders"

Business Requirement
Marketing wants a list of only those customers who have at least one order (to target them with a loyalty campaign). Skip customers with no orders.
SQL Developer Thinks EXISTS returns true if the subquery finds any rows. It's efficient because it stops at the first match. I write WHERE EXISTS (SELECT 1 FROM Orders WHERE...) — the SELECT 1 is just a placeholder since EXISTS only cares about presence, not values.
C# Developer Thinks The .Any() method is C#'s equivalent to EXISTS. No arguments means "does this collection have any items?" Super readable: c.Orders.Any() = "does this customer have any orders?"
SQL — EXISTS
SELECT c.Id, c.Name
FROM Customers c
WHERE EXISTS (
    SELECT 1
    FROM Orders o
    WHERE o.CustomerId = c.Id
);
LINQ — Any()
var customers = await db.Customers
    .AsNoTracking()
    .Where(c => c.Orders.Any())
    .ToListAsync(ct);
LINQ Query Syntax
var customers = await (
    from c in db.Customers
                      .AsNoTracking()
    where c.Orders.Any()
    select c
).ToListAsync(ct);
03

Requirement: "Show Products in These Categories"

Business Requirement
The UI sends a dynamic list of category IDs (e.g., from a multi-select dropdown). Show all products in those categories. The list of IDs can change, so we use IN with a subquery.
SQL Developer Thinks IN checks if a value matches any item in a list. The subquery generates that list dynamically. WHERE CategoryId IN (SELECT Id FROM Categories WHERE IsActive = 1) — super efficient for large lists.
C# Developer Thinks .Contains() is C#'s IN. I pass a collection of IDs and it checks membership. If the IDs are already loaded in memory, it's straightforward. If they're in the database, I build a subquery first.
SQL — IN with Subquery
SELECT p.Id, p.Name, p.Price
FROM Products p
WHERE p.CategoryId IN (
    SELECT c.Id
    FROM Categories c
    WHERE c.IsActive = 1
);
C# — Contains
var activeIds = await db.Categories
    .AsNoTracking()
    .Where(c => c.IsActive)
    .Select(c => c.Id)
    .ToListAsync(ct);

var products = await db.Products
    .AsNoTracking()
    .Where(p => activeIds.Contains(p.CategoryId))
    .ToListAsync(ct);
Dapper — IN with Parameters
int[] categoryIds = { 1, 3, 5 };
var products = await conn.QueryAsync<Product>(
    @"SELECT Id, Name, Price
      FROM Products
      WHERE CategoryId IN (SELECT @Ids)",
    new { Ids = categoryIds });
04

Requirement: "Find Customers with No Orders"

Business Requirement
Re-engagement campaign: find customers who have never placed an order. We need the opposite of EXISTS — customers where no orders exist for them.
SQL Developer Thinks NOT EXISTS is the negation of EXISTS. WHERE NOT EXISTS (SELECT 1 FROM Orders WHERE...) returns true when the subquery finds zero rows. This is the cleanest way to express "no matching orders."
C# Developer Thinks !Any() negates the Any() condition. c.Orders.Any() asks "do you have orders?" — !c.Orders.Any() asks "do you have NO orders?" The exclamation mark is the NOT operator.
SQL — NOT EXISTS
SELECT c.Id, c.Name
FROM Customers c
WHERE NOT EXISTS (
    SELECT 1
    FROM Orders o
    WHERE o.CustomerId = c.Id
);
C# — !Any()
var customers = await db.Customers
    .AsNoTracking()
    .Where(c => !c.Orders.Any())
    .ToListAsync(ct);
Alternative: LEFT JOIN WHERE NULL
SELECT c.Id, c.Name
FROM Customers c
LEFT JOIN Orders o ON c.Id = o.CustomerId
WHERE o.Id IS NULL;
05

Requirement: "Find Products Priced Above Category Average"

Business Requirement
Flag premium products — those that are priced above the average price in their category. This requires a correlated subquery that references the outer query's row.
SQL Developer Thinks A correlated subquery references the outer query's table (p.CategoryId in the subquery references the outer p). For each product, I calculate the average price in that product's category, then compare. Warning: these can be slow on big datasets because the subquery runs once per row.
C# Developer Thinks Direct correlated subqueries in LINQ are tricky. Better approach: load the category averages first with GroupBy, then filter the products. Or use raw SQL with EF.FromSql() for the correlated query.
SQL — Correlated Subquery
SELECT p.Id, p.Name, p.Price
FROM Products p
WHERE p.Price > (
    SELECT AVG(p2.Price)
    FROM Products p2
    WHERE p2.CategoryId = p.CategoryId
);
C# — Two-Step Approach
var avgByCategory = await db.Products
    .AsNoTracking()
    .GroupBy(p => p.CategoryId)
    .Select(g => new { CatId = g.Key,
        AvgPrice = g.Average(p => p.Price) })
    .ToListAsync(ct);

var premium = db.Products
    .AsNoTracking()
    .Where(p => p.Price >
        avgByCategory
        .First(a => a.CatId == p.CategoryId)
        .AvgPrice)
    .ToList();
C# — EF Raw SQL for Correlated Query
var premium = await db.Products
    .FromSql($@"SELECT p.Id, p.Name, p.Price,
                      p.CategoryId
               FROM Products p
               WHERE p.Price > (
                   SELECT AVG(p2.Price)
                   FROM Products p2
                   WHERE p2.CategoryId = p.CategoryId
               )")
    .AsNoTracking()
    .ToListAsync(ct);
06

Requirement: "Build a Report from Aggregated Data"

Business Requirement
Show customers with high average order values. Calculate average order value per customer, then find those above $100. This requires a derived table — a subquery in the FROM clause.
SQL Developer Thinks The subquery in the FROM clause is called a "derived table." It's a virtual table created by the subquery. I name it with AS and then query it like a normal table. First aggregate, then filter on the aggregated data.
C# Developer Thinks LINQ query composition achieves the same result. I build one query, then filter it. The first query groups and aggregates, the second filters on that aggregated result. No special syntax needed.
SQL — Derived Table
SELECT sub.CustomerId, sub.AvgOrder
FROM (
    SELECT CustomerId,
           AVG(Total) AS AvgOrder
    FROM Orders
    GROUP BY CustomerId
) AS sub
WHERE sub.AvgOrder > 100;
C# — Query Composition
var report = await db.Orders
    .AsNoTracking()
    .GroupBy(o => o.CustomerId)
    .Select(g => new
    {
        CustomerId = g.Key,
        AvgOrder = g.Average(o => o.Total)
    })
    .Where(x => x.AvgOrder > 100)
    .ToListAsync(ct);
07

Requirement: "Every Order Must Be Over $10"

Business Requirement
Find customers where ALL their orders are above $10 (i.e., no customer has any order under $10). This is different from "at least one order over $10" — we need ALL orders to qualify.
SQL Developer Thinks The ALL operator compares a value against all values in a subquery. WHERE NOT EXISTS (SELECT 1 FROM Orders WHERE Total <= 10 AND CustomerId = c.Id) means "there do not exist any small orders for this customer." Alternatively, use ALL: WHERE Price > ALL (SELECT ...).
C# Developer Thinks .All() is the counterpart to SQL's ALL. c.Orders.All(o => o.Total > 10) asks "do all of this customer's orders exceed $10?" If ANY order is <= 10, it returns false.
SQL — NOT EXISTS or ALL
SELECT c.Id, c.Name
FROM Customers c
WHERE NOT EXISTS (
    SELECT 1
    FROM Orders o
    WHERE o.Total <= 10
      AND o.CustomerId = c.Id
);
C# — All()
var customers = await db.Customers
    .AsNoTracking()
    .Where(c => c.Orders
                    .All(o => o.Total > 10))
    .ToListAsync(ct);
