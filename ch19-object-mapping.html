Chapter 19

Object Mapping with Dapper

Map SQL results to C# objects by property name convention, handle column name mismatches, and work with records, structs, and dynamic objects.

01

Convention-Based Mapping

Dapper matches SQL column names to C# property names (case-insensitive). The SQL column name must exactly match the property name.

SQL
SELECT Id, FirstName, LastName, Email
FROM Customers
C#
public class Customer
{
    public int Id { get; set; }
    public string FirstName { get; set; }
    public string LastName { get; set; }
    public string Email { get; set; }
}
Matching rule: Column names are matched case-insensitively to property names. FirstName in SQL maps to FirstName in C#.
02

Handling Column Name Mismatches

When SQL column names don't match C# properties, use SQL aliases to rename columns.

SQL with Aliases
SELECT user_id AS Id,
       first_name AS FirstName,
       last_name AS LastName
FROM users
C#
var customers = await connection.QueryAsync<Customer>(sql);

public class Customer
{
    public int Id { get; set; }
    public string FirstName { get; set; }
    public string LastName { get; set; }
}
Aliasing: Always use AS PropertyName in your SQL query to match C# property names exactly.
03

Records and Readonly Structs

Dapper supports records (C# 9+) and structs for immutable mapping targets. Perfect for read-only query results.

C# — Record Class
public record Customer(int Id, string FirstName, string LastName, string Email);

var customers = await connection.QueryAsync<Customer>(sql);
// Results are immutable records
C# — Readonly Struct
public readonly struct CustomerInfo
{
    public int Id { get; init; }
    public string FirstName { get; init; }
    public string LastName { get; init; }
}

var customers = await connection.QueryAsync<CustomerInfo>(sql);
When to use: Records for immutable DTOs. Structs for value-type semantics or memory efficiency in large collections.
04

Dynamic Results

Query without defining a class using Query<dynamic>. Each row is a DynamicObject with properties matching column names.

C#
var results = await connection.QueryAsync<dynamic>(
    "SELECT Id, FirstName, LastName FROM Customers");

foreach (dynamic customer in results)
{
    Console.WriteLine($"{customer.FirstName} {customer.LastName}");
    int id = (int)customer.Id;  // Must cast primitive types
}
Dynamic trade-off: Flexible but no compile-time type safety. Requires casting for primitives. Use for exploratory queries only.
05

Multiple Result Sets

Use QueryMultiple to execute multiple SELECT statements in one round trip and get multiple result sets back.

SQL
SELECT * FROM Customers;
SELECT * FROM Orders;
C#
var sql = @"SELECT * FROM Customers;
            SELECT * FROM Orders;";

using var multi = await connection.QueryMultipleAsync(sql);
var customers = await multi.ReadAsync<Customer>();
var orders = await multi.ReadAsync<Order>();
Efficiency: One network round trip for multiple queries. Each Read<T>() consumes the next result set in order.
06

Custom Type Handlers

Handle custom types or unsupported database types with SqlMapper.AddTypeHandler<T>.

C# — Custom Type Handler
// Register a handler for a custom type
SqlMapper.AddTypeHandler<MyCustomType>(
    parse: (value) => {
        // Convert from DB value (object) to MyCustomType
        return new MyCustomType((string)value);
    },
    setNull: (cmd, param, dbType) => {
        cmd.Parameters.Add(param).Value = DBNull.Value;
    }
);

// Now queries work transparently
var result = connection.Query<MyClass>("SELECT ... FROM ...").ToList();
Use case: Map database JSON/XML to C# objects, or handle database-specific types like PostgreSQL enums.

Key Takeaways

  • Dapper maps columns to properties by name (case-insensitive).
  • Use SQL aliases (AS PropertyName) when column names don't match properties.
  • Support for records, structs, and classes — choose based on immutability needs.
  • Query<dynamic> for schemaless results, but lose compile-time safety.
  • QueryMultiple combines multiple SELECT statements in one round trip.
  • Custom type handlers map unsupported database types to C# types.
