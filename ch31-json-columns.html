JSON Columns

Store and query JSON in SQL Server, PostgreSQL. Semi-structured data without extra tables.

In This Chapter

01

JSON in SQL

Store nested data as JSON without creating extra tables.

SQL Server: NVARCHAR(MAX) CREATE TABLE Customers ( Id INT PRIMARY KEY, Name NVARCHAR(100), Metadata NVARCHAR(MAX) -- JSON string ); INSERT INTO Customers VALUES (1, 'Alice', '{"phone":"555-1234","address":"123 Main St","tags":["vip","premium"]}'); -- Query JSON SELECT Id, Name, JSON_VALUE(Metadata, '$.phone') AS Phone, JSON_QUERY(Metadata, '$.tags') AS Tags FROM Customers;
PostgreSQL: jsonb Type CREATE TABLE customers ( id SERIAL PRIMARY KEY, name VARCHAR(100), metadata JSONB -- Native JSON type ); INSERT INTO customers VALUES (1, 'Alice', '{"phone":"555-1234","address":"123 Main St","tags":["vip","premium"]}'::jsonb); -- Query JSON SELECT id, name, metadata->>'phone' AS phone, metadata->'tags' AS tags FROM customers;
SQL Server vs PostgreSQL:

SQL Server: use NVARCHAR(MAX), parse with JSON_VALUE/JSON_QUERY. PostgreSQL: use jsonb, supports indexing.

02

EF Core Value Conversion

Map C# objects to JSON columns automatically.

Define Domain Model public class Customer { public int Id { get; set; } public string Name { get; set; } // Complex type stored as JSON public CustomerMetadata Metadata { get; set; } } public class CustomerMetadata { public string Phone { get; set; } public string Address { get; set; } public List<string> Tags { get; set; } } // Configuration modelBuilder.Entity<Customer>(e => { e.Property(c => c.Metadata) .HasConversion( v => JsonConvert.SerializeObject(v), // C# to JSON v => JsonConvert.DeserializeObject<CustomerMetadata>(v)); // JSON to C# // EF Core handles serialization automatically });
Alternative:

EF Core 7+ supports System.Text.Json natively without explicit conversion.

03

Querying JSON

Query into JSON columns using EF functions.

Filter by JSON Properties // Get customers with 'vip' tag var customers = await db.Customers .AsNoTracking() .Where(c => EF.Functions.JsonContains( c.Metadata, "vip")) .ToListAsync(); // Alternative: extract and filter var customers = await db.Customers .AsNoTracking() .Where(c => c.Metadata.Tags.Contains("vip")) .ToListAsync(); // EF translates to JSON_CONTAINS or jsonb operators
04

When to Use JSON

JSON columns solve specific problems. Not a replacement for normalization.

ScenarioUse JSON?
Metadata (tags, settings)Yes. Schema flexible.
Audit trail (immutable history)Yes. Array of events.
Related records you always query togetherMaybe. Depends on queries.
Many-to-many relationshipsNo. Use junction table.
Frequently filtered columnsNo. Use proper columns + index.
Rule:

JSON for semi-structured, query-rarely data. Relational tables for frequently filtered, normalized data.
