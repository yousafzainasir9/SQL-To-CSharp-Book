Chapter 21

Stored Procedures with Dapper

Call SQL Server stored procedures from C#, pass input parameters, retrieve output parameters and return values, and handle result sets.

01

Calling a Stored Procedure

Use QueryAsync or ExecuteAsync with CommandType.StoredProcedure to call a stored procedure.

SQL
CREATE PROCEDURE sp_GetCustomer
    @CustomerId INT
AS
BEGIN
    SELECT Id, FirstName, LastName, Email
    FROM Customers
    WHERE Id = @CustomerId
END
C#
var customer = await connection.QueryFirstOrDefaultAsync<Customer>(
    "sp_GetCustomer",
    new { CustomerId = 1 },
    commandType: CommandType.StoredProcedure);
Key: Set commandType: CommandType.StoredProcedure. Procedure name and parameters work the same as inline SQL.
02

Input Parameters

Pass parameters as an anonymous object. Dapper automatically maps property names to procedure parameters.

SQL
CREATE PROCEDURE sp_SearchCustomers
    @FirstName NVARCHAR(100),
    @LastName NVARCHAR(100),
    @City NVARCHAR(100)
AS
BEGIN
    SELECT * FROM Customers
    WHERE FirstName LIKE '%' + @FirstName + '%'
      AND LastName LIKE '%' + @LastName + '%'
      AND City LIKE '%' + @City + '%'
END
C#
var results = await connection.QueryAsync<Customer>(
    "sp_SearchCustomers",
    new { FirstName = "John", LastName = "Doe", City = "Seattle" },
    commandType: CommandType.StoredProcedure);
Matching: Property names must match procedure parameter names (case-insensitive). Use null for optional parameters.
03

Output Parameters

Use DynamicParameters to pass output parameters. Retrieve values after execution.

SQL
CREATE PROCEDURE sp_CreateCustomer
    @FirstName NVARCHAR(100),
    @LastName NVARCHAR(100),
    @NewCustomerId INT OUTPUT
AS
BEGIN
    INSERT INTO Customers (FirstName, LastName)
    VALUES (@FirstName, @LastName)
    SET @NewCustomerId = SCOPE_IDENTITY()
END
C#
var parameters = new DynamicParameters();
parameters.Add("@FirstName", "Alice");
parameters.Add("@LastName", "Smith");
parameters.Add("@NewCustomerId", dbType: DbType.Int32, direction: ParameterDirection.Output);

await connection.ExecuteAsync(
    "sp_CreateCustomer",
    parameters,
    commandType: CommandType.StoredProcedure);

int newId = parameters.Get<int>("@NewCustomerId");
Output flow: Define parameter with direction: ParameterDirection.Output. After execution, retrieve with Get<T>.
04

Return Values

Capture a procedure's return value (success/error code) using ParameterDirection.ReturnValue.

SQL
CREATE PROCEDURE sp_DeleteCustomer
    @CustomerId INT
AS
BEGIN
    IF EXISTS (SELECT 1 FROM Customers WHERE Id = @CustomerId)
    BEGIN
        DELETE FROM Customers WHERE Id = @CustomerId
        RETURN 0  -- Success
    END
    ELSE
    BEGIN
        RETURN 1  -- Not found
    END
END
C#
var parameters = new DynamicParameters();
parameters.Add("@CustomerId", 1);
parameters.Add("@returnValue", dbType: DbType.Int32, direction: ParameterDirection.ReturnValue);

await connection.ExecuteAsync(
    "sp_DeleteCustomer",
    parameters,
    commandType: CommandType.StoredProcedure);

int returnValue = parameters.Get<int>("@returnValue");
if (returnValue is 0)
    Console.WriteLine("Deleted successfully");
else
    Console.WriteLine("Customer not found");
Return value: Always an int. Use for error codes or status indicators.
05

Multiple Result Sets

A stored procedure can return multiple SELECT result sets. Use QueryMultiple to retrieve them all.

SQL
CREATE PROCEDURE sp_GetCustomerDetails
    @CustomerId INT
AS
BEGIN
    SELECT * FROM Customers WHERE Id = @CustomerId
    SELECT * FROM Orders WHERE CustomerId = @CustomerId
    SELECT * FROM OrderItems WHERE OrderId IN
        (SELECT Id FROM Orders WHERE CustomerId = @CustomerId)
END
C#
using var multi = await connection.QueryMultipleAsync(
    "sp_GetCustomerDetails",
    new { CustomerId = 1 },
    commandType: CommandType.StoredProcedure);
var customer = await multi.ReadFirstAsync<Customer>();
var orders = await multi.ReadAsync<Order>();
var items = await multi.ReadAsync<OrderItem>();
Order matters: Each Read<T> consumes the next SELECT result set in order.
06

Async Procedure Calls

Use async variants like QueryAsync, ExecuteAsync, and QueryMultipleAsync for non-blocking calls.

C#
public async Task<Customer> GetCustomerAsync(int customerId, CancellationToken cancellationToken = default)
{
    await using var connection = new SqlConnection(connectionString);
    return await connection.QueryFirstOrDefaultAsync<Customer>(
        "sp_GetCustomer",
        new { CustomerId = customerId },
        commandType: CommandType.StoredProcedure);
}

public async Task<int> CreateCustomerAsync(string firstName, string lastName, CancellationToken cancellationToken = default)
{
    await using var connection = new SqlConnection(connectionString);
    var parameters = new DynamicParameters();
    parameters.Add("@FirstName", firstName);
    parameters.Add("@LastName", lastName);
    parameters.Add("@NewCustomerId", dbType: DbType.Int32,
        direction: ParameterDirection.Output);

    await connection.ExecuteAsync(
        "sp_CreateCustomer",
        parameters,
        commandType: CommandType.StoredProcedure);

    return parameters.Get<int>("@NewCustomerId");
}
Async pattern: All methods have Async variants. Use for I/O-bound operations to avoid blocking threads.

Key Takeaways

  • Set commandType: CommandType.StoredProcedure to call stored procedures.
  • Pass input parameters as an anonymous object. Names match procedure parameters.
  • DynamicParameters handles output parameters and return values.
  • Retrieve output with parameters.Get<T>("@ParameterName").
  • QueryMultiple retrieves multiple result sets from one procedure call.
  • Use async variants (QueryAsync, ExecuteAsync) for non-blocking calls.
