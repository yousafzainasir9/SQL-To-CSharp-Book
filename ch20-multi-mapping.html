Chapter 20

Multi-Mapping with Dapper

Requirement: "Show orders with customer details — in one query." In EF, you use Include(). In Dapper, you use multi-mapping: one SQL JOIN, and Dapper splits the result row into multiple objects using splitOn.

01

Requirement: "Show Each Order with Its Customer"

Business Requirement
Order list page shows order details + customer name in one query. No N+1 queries. One trip to the database.
SQL Developer Thinks I'll JOIN Orders to Customers. SELECT o.*, c.* FROM Orders o JOIN Customers c ON o.CustomerId = c.Id. The database returns one row per order, with both order and customer columns mixed together.
C# Developer Thinks I need to tell Dapper: "The first set of columns (o.*) maps to Order. Then at the Id column, start mapping to Customer (c.*)." That's what splitOn: "Id" does. Dapper reads column positions and knows where the split happens.
SQL — JOIN
SELECT
    o.Id, o.OrderDate, o.Total, o.CustomerId,
    c.Id, c.Name, c.Email
FROM Orders o
JOIN Customers c ON o.CustomerId = c.Id
WHERE o.OrderDate >= @StartDate
ORDER BY o.OrderDate DESC;
Dapper — Multi-Mapping
await using var conn = new SqlConnection(cs);

var orders = await conn.QueryAsync<Order, Customer, Order>(
    @"SELECT o.Id, o.OrderDate, o.Total,
             o.CustomerId,
             c.Id, c.Name, c.Email
      FROM Orders o
      JOIN Customers c ON o.CustomerId = c.Id
      WHERE o.OrderDate >= @Start",
    (order, customer) =>
    {
        order.Customer = customer;
        return order;
    },
    new { Start = startDate },
    splitOn: "Id");

return orders.ToList();
C# — EF Include for Comparison
var orders = await db.Orders
    .AsNoTracking()
    .Include(o => o.Customer)
    .Where(o => o.OrderDate >= startDate)
    .OrderByDescending(o => o.OrderDate)
    .ToListAsync(ct);
02

Requirement: "Show Each Customer with All Their Orders"

Business Requirement
Customer detail page lists all orders per customer. The JOIN returns one row per order, so the customer data repeats. We need to deduplicate and group orders under each customer.
SQL Developer Thinks SELECT c.*, o.* FROM Customers c LEFT JOIN Orders o ON c.Id = o.CustomerId ORDER BY c.Id. This returns one row per order, repeating the customer info. The app must group them.
C# Developer Thinks I'll map both objects, then use a Dictionary to group: iterate the results, accumulate orders under each CustomerId, and skip duplicate customer rows. The final result is a list of customers with their orders nested.
SQL — LEFT JOIN (Repeats Customer for Each Order)
SELECT
    c.Id, c.Name, c.Email, c.Phone,
    o.Id, o.OrderDate, o.Total
FROM Customers c
LEFT JOIN Orders o ON c.Id = o.CustomerId
ORDER BY c.Id, o.OrderDate;
Dapper — Multi-Map + Dictionary Grouping
await using var conn = new SqlConnection(cs);

var customerDict = new Dictionary<int, Customer>();

await conn.QueryAsync<Customer, Order, Customer>(
    @"SELECT c.Id, c.Name, c.Email, c.Phone,
             o.Id, o.OrderDate, o.Total
      FROM Customers c
      LEFT JOIN Orders o ON c.Id = o.CustomerId
      ORDER BY c.Id, o.OrderDate",
    (customer, order) =>
    {
        if (!customerDict.TryGetValue(customer.Id, out var cust))
        {
            cust = customer;
            cust.Orders = new List<Order>();
            customerDict.Add(cust.Id, cust);
        }

        if (order != null)
            cust.Orders.Add(order);

        return cust;
    },
    splitOn: "Id");

return customerDict.Values.ToList();
C# Entity — Customer with Orders Collection
public class Customer
{
    public int Id { get; set; }
    public string Name { get; set; } = "";
    public string Email { get; set; } = "";
    public string Phone { get; set; } = "";

    // Navigation — all orders for this customer
    public List<Order> Orders { get; set; } = new();
}

public class Order
{
    public int Id { get; set; }
    public DateTime OrderDate { get; set; }
    public decimal Total { get; set; }
    public int CustomerId { get; set; }
}
03

Requirement: "Show Order with Customer, Product, and Category"

Business Requirement
Invoice page needs Order + Customer + OrderItem (line items) + Product + Category info. Chain multiple JOINs and map to 4 different types.
SQL Developer Thinks Multiple JOINs: Orders → Customers, Orders → OrderItems → Products → Categories. The result set is wider — more columns to map.
C# Developer Thinks QueryAsync<T1, T2, T3, T4, TReturn> with a map function that receives all 4 objects. Use splitOn multiple times: "Id,Id,Id" to mark where each object boundary starts.
SQL — Multiple JOINs
SELECT
    o.Id, o.OrderDate, o.Total, o.CustomerId,
    c.Id, c.Name, c.Email,
    oi.Id, oi.Quantity, oi.UnitPrice,
    p.Id, p.Name AS ProductName, p.Sku,
    cat.Id, cat.Name AS CategoryName
FROM Orders o
JOIN Customers c ON o.CustomerId = c.Id
JOIN OrderItems oi ON o.Id = oi.OrderId
JOIN Products p ON oi.ProductId = p.Id
JOIN Categories cat ON p.CategoryId = cat.Id
WHERE o.Id = @OrderId
ORDER BY oi.Id;
Dapper — QueryAsync with 4 Types
await using var conn = new SqlConnection(cs);

var invoiceDict = new Dictionary<int, Order>();
var itemDict = new Dictionary<int, OrderItem>();

await conn.QueryAsync<Order, Customer, OrderItem, Product, Category, Order>(
    @"SELECT o.Id, o.OrderDate, o.Total, o.CustomerId,
             c.Id, c.Name, c.Email,
             oi.Id, oi.Quantity, oi.UnitPrice,
             p.Id, p.Name, p.Sku,
             cat.Id, cat.Name
      FROM Orders o
      JOIN Customers c ON o.CustomerId = c.Id
      JOIN OrderItems oi ON o.Id = oi.OrderId
      JOIN Products p ON oi.ProductId = p.Id
      JOIN Categories cat ON p.CategoryId = cat.Id
      WHERE o.Id = @OrderId",
    (order, customer, item, product, category) =>
    {
        if (!invoiceDict.TryGetValue(order.Id, out var ord))
        {
            ord = order;
            ord.Customer = customer;
            ord.Items = new List<OrderItem>();
            invoiceDict.Add(ord.Id, ord);
        }

        if (!itemDict.TryGetValue(item.Id, out var itm))
        {
            itm = item;
            itm.Product = product;
            itm.Product.Category = category;
            ord.Items.Add(itm);
            itemDict.Add(itm.Id, itm);
        }

        return ord;
    },
    new { OrderId = orderId },
    splitOn: "Id,Id,Id,Id");

return invoiceDict.Values.First();
04

Requirement: "Load Two Separate Lists from One Database Call"

Business Requirement
Dashboard needs both customer summary AND recent orders. Instead of two round trips, batch both queries in one call and read two result sets.
SQL Developer Thinks Two SELECT statements in one batch. Separate them by a semicolon or GO. The database sends two result sets back in one response.
C# Developer Thinks QueryMultipleAsync() sends both statements at once. Use gridReader.ReadAsync<T>() to read each result set sequentially. One network call, two lists.
SQL — Batch Query
SELECT TOP 5
    Id, Name, City, TotalOrders = (
        SELECT COUNT(*) FROM Orders WHERE CustomerId = c.Id
    )
FROM Customers c
WHERE IsActive = 1
ORDER BY TotalOrders DESC;

SELECT TOP 10
    o.Id, o.OrderDate, o.Total, c.Name AS CustomerName
FROM Orders o
JOIN Customers c ON o.CustomerId = c.Id
ORDER BY o.OrderDate DESC;
Dapper — QueryMultipleAsync
await using var conn = new SqlConnection(cs);

using (var grid = await conn.QueryMultipleAsync(
    @"SELECT TOP 5 Id, Name, City,
             TotalOrders = (SELECT COUNT(*)
                            FROM Orders
                            WHERE CustomerId = c.Id)
      FROM Customers c
      WHERE IsActive = 1
      ORDER BY TotalOrders DESC;

      SELECT TOP 10 o.Id, o.OrderDate, o.Total,
                    c.Name AS CustomerName
      FROM Orders o
      JOIN Customers c ON o.CustomerId = c.Id
      ORDER BY o.OrderDate DESC;",
    commandTimeout: 30))
{
    var topCustomers = await grid.ReadAsync<CustomerSummary>();
    var recentOrders = await grid.ReadAsync<RecentOrder>();

    return new Dashboard
    {
        TopCustomers = topCustomers.ToList(),
        RecentOrders = recentOrders.ToList()
    };
}
05

Requirement: "Performance — Multi-Map vs Separate Queries"

Business Requirement
Which approach is fastest and uses least memory? Multi-map one query, separate queries with N+1 trips, or QueryMultiple for grouped results?
Approach Queries Network Trips Memory Best For
Multi-Mapping 1 1 High (row duplication) One-to-one, small one-to-many (5–10 children)
Separate Queries N+1 N+1 Low Few parents, clean code preferred
QueryMultiple M 1 Medium Unrelated queries (e.g., dashboard: top 5 & recent 10)
EF Include 1 1 High Auto-deduplication, change tracking needed
SQL Developer Thinks All approaches send the same data from SQL Server to the network. The difference is how the C# app processes and stores it in memory. One query = less latency. Separate queries = more round trips.
C# Developer Thinks Multi-mapping duplicates object data in memory (same customer repeated 100 times for 100 orders). Separate queries = 1 customer query + 100 order queries. QueryMultiple = 2 queries, 1 trip. Pick based on row count and tolerance for network latency.
Dapper — When to Use Each
// ONE-TO-ONE: Use multi-mapping
// Example: Order + Customer
// Result: 1000 rows × (Order + Customer) = 1000 objects

// ONE-TO-MANY (small): Use multi-mapping + Dictionary
// Example: Customer + Orders (3 orders per customer)
// Result: 300 rows, grouped into 100 customers + orders

// ONE-TO-MANY (large): Use separate queries
// Example: Customer + Orders (100 per customer = 10K rows)
// Result: 1 customer query + 100 order queries = 101 calls

// UNRELATED SETS: Use QueryMultiple
// Example: Top 5 customers + Recent 10 orders
// Result: 2 queries, 1 network trip
