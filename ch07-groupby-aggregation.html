Chapter 07

GROUP BY & Aggregation

You get a requirement: "How many orders does each customer have?" A SQL developer writes GROUP BY with COUNT. A C# developer writes .GroupBy() with .Count(). Both aggregate rows into summaries.

01

Requirement: "Count Orders Per Customer"

Business Requirement
Dashboard shows each customer with their total order count. We need to see every customer and how many orders they’ve placed.
SQL Developer Thinks GROUP BY customer_id, then COUNT(*) to aggregate rows. The GROUP BY organizes the data into buckets, one per customer. COUNT(*) counts how many orders are in each bucket.
C# Developer Thinks .GroupBy(o => o.CustomerId) creates groups. Each group has a .Key (the CustomerId) and a collection of orders. .Count() counts items in each group. Or use .Select() to project the result.
SQL — GROUP BY
SELECT
    CustomerId,
    COUNT(*) AS OrderCount
FROM Orders
GROUP BY CustomerId
ORDER BY OrderCount DESC;
LINQ — Method Syntax
var results = await db.Orders
    .AsNoTracking()
    .GroupBy(o => o.CustomerId)
    .Select(g => new
    {
        CustomerId = g.Key,
        OrderCount = g.Count()
    })
    .OrderByDescending(x => x.OrderCount)
    .ToListAsync(ct);
LINQ — Query Syntax
var results = await (
    from o in db.Orders.AsNoTracking()
    group o by o.CustomerId into g
    orderby g.Count() descending
    select new
    {
        CustomerId = g.Key,
        OrderCount = g.Count()
    }
).ToListAsync(ct);
02

Requirement: "Show Revenue by Category"

Business Requirement
Finance needs total revenue, average order value, highest and lowest order per product category. We need all four aggregate functions together.
SQL Developer Thinks GROUP BY category_id, then SUM(total) for revenue, AVG(total) for average, MAX(total) for highest, MIN(total) for lowest. All five aggregates in one query.
C# Developer Thinks .GroupBy(o => o.Category). Inside .Select(), calculate all aggregates on the group: g.Sum(), g.Average(), g.Max(), g.Min(). Project to a new object with all computed values.
SQL — All Aggregates
SELECT
    c.Id,
    c.Name,
    SUM(o.Total) AS TotalRevenue,
    AVG(o.Total) AS AvgOrder,
    MAX(o.Total) AS HighestOrder,
    MIN(o.Total) AS LowestOrder
FROM Orders o
JOIN Categories c ON o.CategoryId = c.Id
GROUP BY c.Id, c.Name;
LINQ — Method Syntax
var results = await db.Orders
    .AsNoTracking()
    .GroupBy(o => new { o.Category.Id, o.Category.Name })
    .Select(g => new
    {
        g.Key.Id,
        g.Key.Name,
        TotalRevenue = g.Sum(o => o.Total),
        AvgOrder = g.Average(o => o.Total),
        HighestOrder = g.Max(o => o.Total),
        LowestOrder = g.Min(o => o.Total)
    })
    .ToListAsync(ct);
LINQ — Query Syntax
var results = await (
    from o in db.Orders.AsNoTracking()
    group o by new { o.Category.Id, o.Category.Name } into g
    select new
    {
        g.Key.Id,
        g.Key.Name,
        TotalRevenue = g.Sum(o => o.Total),
        AvgOrder = g.Average(o => o.Total),
        HighestOrder = g.Max(o => o.Total),
        LowestOrder = g.Min(o => o.Total)
    }
).ToListAsync(ct);
03

Requirement: "Only Show Categories with 10+ Products"

Business Requirement
Filter out small categories — only show categories that have at least 10 products in stock. This filter happens AFTER grouping, not before.
SQL Developer Thinks WHERE filters BEFORE grouping. HAVING filters AFTER grouping. Since I need to filter on a COUNT, I use HAVING COUNT(*) >= 10, not WHERE.
C# Developer Thinks .GroupBy().Select() creates the groups and aggregates. Then .Where() on the GROUP RESULT filters out groups that don’t meet the criterion. It’s Where on the grouped collection.
SQL — HAVING Clause
SELECT
    CategoryId,
    COUNT(*) AS ProductCount
FROM Products
GROUP BY CategoryId
HAVING COUNT(*) >= 10
ORDER BY ProductCount DESC;
LINQ — Method Syntax
var results = await db.Products
    .AsNoTracking()
    .GroupBy(p => p.CategoryId)
    .Where(g => g.Count() >= 10)
    .Select(g => new
    {
        CategoryId = g.Key,
        ProductCount = g.Count()
    })
    .OrderByDescending(x => x.ProductCount)
    .ToListAsync(ct);
LINQ — Query Syntax
var results = await (
    from p in db.Products.AsNoTracking()
    group p by p.CategoryId into g
    where g.Count() >= 10
    orderby g.Count() descending
    select new
    {
        CategoryId = g.Key,
        ProductCount = g.Count()
    }
).ToListAsync(ct);
04

Requirement: "Revenue by Year and Category"

Business Requirement
Finance dashboard breaks down revenue by year AND category. We need to group by TWO columns so we can see trends per category across years.
SQL Developer Thinks GROUP BY multiple columns: GROUP BY YEAR(OrderDate), CategoryId. Each row in the result represents one unique combination of year + category.
C# Developer Thinks .GroupBy(o => new { Year = o.OrderDate.Year, o.CategoryId }). The grouping key is now an anonymous object with two properties. The .Key has both values.
SQL — GROUP BY Two Columns
SELECT
    YEAR(o.OrderDate) AS OrderYear,
    o.CategoryId,
    c.Name AS CategoryName,
    SUM(o.Total) AS Revenue
FROM Orders o
JOIN Categories c ON o.CategoryId = c.Id
GROUP BY YEAR(o.OrderDate), o.CategoryId, c.Name
ORDER BY OrderYear DESC, Revenue DESC;
LINQ — Method Syntax
var results = await db.Orders
    .AsNoTracking()
    .GroupBy(o => new
    {
        Year = o.OrderDate.Year,
        o.CategoryId,
        o.Category.Name
    })
    .Select(g => new
    {
        g.Key.Year,
        g.Key.CategoryId,
        g.Key.Name,
        Revenue = g.Sum(o => o.Total)
    })
    .OrderByDescending(x => x.Year)
    .ThenByDescending(x => x.Revenue)
    .ToListAsync(ct);
LINQ — Query Syntax
var results = await (
    from o in db.Orders.AsNoTracking()
    group o by new
    {
        Year = o.OrderDate.Year,
        o.CategoryId,
        o.Category.Name
    } into g
    orderby g.Key.Year descending, g.Sum(o => o.Total) descending
    select new
    {
        g.Key.Year,
        g.Key.CategoryId,
        g.Key.Name,
        Revenue = g.Sum(o => o.Total)
    }
).ToListAsync(ct);
05

Requirement: "Top 3 Customers by Spending"

Business Requirement
VIP list — top 3 customers by total spend. Show customer name, ID, and total revenue. We need aggregate + sort + limit together.
SQL Developer Thinks GROUP BY customerId, SUM(total) to get totals. ORDER BY total DESC to sort from highest to lowest. TOP 3 to limit the result.
C# Developer Thinks .GroupBy() to aggregate, .Select() to shape result, .OrderByDescending() to sort by total, .Take(3) to limit. Chain all four operators.
SQL — TOP with ORDER BY
SELECT TOP 3
    c.Id,
    c.Name,
    SUM(o.Total) AS TotalSpent
FROM Orders o
JOIN Customers c ON o.CustomerId = c.Id
GROUP BY c.Id, c.Name
ORDER BY TotalSpent DESC;
LINQ — Method Syntax
var topCustomers = await db.Orders
    .AsNoTracking()
    .GroupBy(o => new { o.Customer.Id, o.Customer.Name })
    .Select(g => new
    {
        g.Key.Id,
        g.Key.Name,
        TotalSpent = g.Sum(o => o.Total)
    })
    .OrderByDescending(x => x.TotalSpent)
    .Take(3)
    .ToListAsync(ct);
LINQ — Query Syntax
var topCustomers = await (
    from o in db.Orders.AsNoTracking()
    group o by new { o.Customer.Id, o.Customer.Name } into g
    orderby g.Sum(o => o.Total) descending
    select new
    {
        g.Key.Id,
        g.Key.Name,
        TotalSpent = g.Sum(o => o.Total)
    }
).Take(3).ToListAsync(ct);
Dapper — Direct SQL
await using var conn = new SqlConnection(connectionString);
var topCustomers = await conn.QueryAsync<CustomerSpendDto>(
    @"SELECT TOP 3
      c.Id,
      c.Name,
      SUM(o.Total) AS TotalSpent
      FROM Orders o
      JOIN Customers c ON o.CustomerId = c.Id
      GROUP BY c.Id, c.Name
      ORDER BY TotalSpent DESC");

Quick Reference: Aggregate Functions

Common Aggregates
SQL and C# have the same aggregate functions with different syntax. All work the same way: they reduce many rows to a single value.
FunctionSQL SyntaxLINQ MethodPurpose
COUNTCOUNT(*).Count()Count all rows in group
COUNTCOUNT(ColumnName).Count(x => x.Col != null)Count non-null values
COUNT DISTINCTCOUNT(DISTINCT ColumnName).Select(x => x.Col).Distinct().Count()Count unique values
SUMSUM(ColumnName).Sum(x => x.Col)Total of numeric column
AVGAVG(ColumnName).Average(x => x.Col)Average of numeric column
MINMIN(ColumnName).Min(x => x.Col)Smallest value
MAXMAX(ColumnName).Max(x => x.Col)Largest value
SQL Aggregates Only Work in GROUP BY You can only use COUNT, SUM, AVG, etc. in the SELECT if you also have GROUP BY. Or wrap them in a subquery. They reduce groups of rows.
C# Aggregates Work on Any Collection You can call .Count() on any IEnumerable without grouping. You can .Sum() any list. LINQ aggregates are methods on collections, not tied to GROUP BY.
